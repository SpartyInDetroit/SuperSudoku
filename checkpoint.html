List all of the classes you expect your project to require and a brief description of the purpose of the class:

 

Cell – There will be two types of this, one will contain the data for the cell, the answer, guess, and hints/notes. The other will be just a number to display so the user can not change the value. The second type is just for initializing the board.

Board – This will be the current game board

Solution – This will contain the 10 possible board solutions

View – Displays the board

Controller – Take in the input and make decisions

 

Indicate how you will encode a sample puzzle as PHP. Remember, that you need to encode not only how the puzzle will appear to the user, but the solution as well.

There will be two boards, the Solution board and the Game board. Both boards will be an array of 9 arrays, creating a 9x9 board. There will be two types of elements in the arrays, one being a cell which will have the answer, the user’s guess, and an array of 9 possible notes so the user can keep track of possible solutions in the cell. The other will be just a number. This distinction is made because when the game is initialized, the starting board is not allowed to be changed. When the new game is created, the Solution board will fill in the answer for each cell of the Game board to check the final answer.
